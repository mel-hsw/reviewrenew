# Data directory

This folder separates **raw inputs**, **derived catalog**, and **ephemeral indexes** so the project stays reproducible and easy to grade (**scientific rigor**: clear provenance, no mystery files).

## Layout

| Path | Role | Version control |
|------|------|-----------------|
| **`raw/amazon_reviews_2023/`** | [**Amazon Reviews 2023**](https://amazon-reviews-2023.github.io/) JSONL **slices** (meta + reviews per category) used by `scripts/consolidate_products.py` | **`*.jsonl` is gitignored** in this repo (large files). Download slices locally or run `git add -f data/raw/amazon_reviews_2023/<file>.jsonl` if graders require them in Git. |
| **`products.json`** | **Derived** three-product catalog (listing text, cleaned reviews, image URLs) consumed by `run_pipeline.py` | Typically committed so TAs can rerun without raw slices. |
| **`products_manifest.json`** | Provenance from consolidation (sources, row counts, timestamps) | Commit alongside `products.json` when using the script. |
| **`products.example.json`** | Minimal template for hand-built catalogs | Commit. |
| **`chroma/`** | Persistent **Chroma** vector index (embeddings). Regenerated by the pipeline. | **Gitignored** — large, machine-local, safe to delete to force re-embed. |

## Hygiene (rubric-aligned)

- **Deliverables** ask for **links or files** for data collected: point readers here and, if raw files are huge, add a **public dataset link** + exact **category / ASIN** selection in the report (`submission/README.md` checklist).
- **Do not commit** secrets (`.env`), API chat logs, or screenshots containing keys.
- After changing **`products.json`** or **chunking** env vars, delete **`chroma/`** before the next run so retrieval matches the corpus.

## Data provenance

- **Primary source:** [Amazon Reviews 2023](https://amazon-reviews-2023.github.io/) (McAuley Lab; documentation, downloads, and citation on that site).
- **What we use:** category JSONL exports (reviews + item metadata) placed under `raw/amazon_reviews_2023/`, then consolidated into `products.json` for this course project.
- **Citation (from the dataset homepage):** Hou et al., *Bridging Language and Items for Retrieval and Recommendation*, arXiv:2403.03952, 2024 — see the site for the authoritative BibTeX.

Credit the dataset URL and paper in your **final report** and any public fork of this repo.

## Dataset terms

Use the reviews corpus in line with the **license / terms** published with [Amazon Reviews 2023](https://amazon-reviews-2023.github.io/), your institution’s policies, and your course’s fair-use rules. Listing text and images may additionally be subject to marketplace terms when reproduced in coursework.
